Elements 1 - Elements vs Tags

4

Now that you know a little about layouts, you can start to learn what elements make up a website. We'll start with the base elements that every website must have and grow from there.

What is an element? Is an element a tag?

An element is the entire statement. An opening tag, the internal items, and a closing tag makes up an element. This is a full element:

<h1 class="large-text">underscore_kb</h1>

A tag is only the opening or closing section of an element. These are the tags in the above elements:

<h1> </h1>

Elements 2 - Doctype

5

The first element we need to discuss is the doctype element. But, this isn't actually an element.

<!DOCTYPE html>

The doctype element is actually a declaration. By using this statement, you are telling the browser that type of document it is reading.

You can read more about the doctype declaration here. If you are using XHTML, you must declare a proper doctype declaration to instruct the browser of the special intention. For most websites, unless you plan on using XHTML specifically, you are free to use the html declaration, as shown above.

Elements 3 - The Comment

6

The comment element is used to document code. It is completely ignored when the browser processes the document.

<!--This is a comment.-->

This element is used to add a comment to the source code. The browser processing the document will completely ignore the comment. Developers use comments to explain code. They are often used to keep up with what a developer was working on when they had to step away.

Comments are useful when debugging code. Commenting out a piece of code can help you understand exactly what that piece of code is doing. For instance, if you have a paragraph element that has a larger font size than you'd expect, you can comment out all of the CSS that is changing font size one by one until you figure out which is causing the change.

The only other thing you need to know about commenting is that you can use it to to hide scripts from browsers without support for scripts. I was not aware of this prior to this writing. You can use the comment element to "hide" script syntax from the browser in case the browser doesn't have the ability to process the script. This is shown below.

<script type="text/javascript"> <!-- function displayMsg() { alert("Hello World!") } --> </script>

Elements 4 - The HTML Element

7

The HTML element is pretty clear cut. It tells the browser that the document is an HTML document. It represents the root of an HTML document. This means that it is the beginning of the document. It should wrap all the other elements on your website except the DOCTYPE declaration. The HTML element should always follow the DOCTYPE declaration like this:

<!DOCTYPE html> <html> ...other elements... </html>

Any div, p, or span elements, and all other elements, should be inside the html element. The only exception to this rule is the DOCTYPE declaration. It will always be outside the html element, as shown above.

The html element is also used for XHTML. I don't fully understand this, so I won't pretend to here for now. You can read more about it here if you're into XHTML. Yes. This is a cop out. I'm hoping to be able to tell you more about XHTML one day.

The html element can be styled with CSS. It may feel tempting to assign a background color to the html element with CSS, but this isn't a good idea. My mentors and the tutorials I've watched say this is a common rookie mistake. You often don't want your entire document to have a background color, but rather you want to build an element inside the HTML element that represents the background. More on this later.

For now, all you really need to know is that browsers have default styles for the html element that may cause things to not work the way you expect them to. You can reset these styles in your CSS document. This topic is better suited for the CSS page, so I'm going to leave it at that.

Elements 5 - The Head Element

8

The Head element is a container for elements that must go inside the head.

Elements that go inside a head element. <title></title> (required in an HTML document) <style></style> (not required to be in the head) <base> <link> <meta> <script></script> (not required to be in the head) <noscript></noscript>

The head element was required in HTML 4.01, but in HTML5 it can be left out. For the most part, all you need to know is that you put the above elements inside the head element. You will almost definitely do this with the link and style elements. Title is required and must be in the head element, so you'll definitely do that.

Elements 6 - The Body Element

9

The body element encapsulates all of the display elements of the website. All of your images, links, paragraphs, articles, navigation buttons, etc will be inside the body element.

The default CSS for most browsers for the body element is:

body { display: block; margin: 8px; } body:focus { outline: none; }

The HTML DOM is an object model for HTML. It defines HTML elements as objects. It stores properties, methods, and events for all HTML elements.

Global Attributes

11

Now that we've discussed the DOM, we should touch on Global Attributes.

Global Attributes are an important part of web development. Arguably, CSS would not work well at all without these attributes.

That is mostly because the "class" attribute is part of global attributes. In CSS, when a developer wants to assign styling to all elements in a single group, the developer will assign a class to each of those elements. The class Global Attribute can then be referenced from a CSS sheet and all elements with that class will be changed by the styling.

There are a lot of Global Attributes. I won't outline them all here, but you can go over the list for yourself at the w3schools website.

Elements 7 - The Title Element

13

The next few elements are all elements that should be placed inside the head element most of the time. Some of these elements are exceptions. I'll highlight them.

The title element defines the title of the website. Whatever you put in this element will display on the tab of the page in the browser. For instance, if you look at the tab for this page, it should say "_kb - HTML". That is because I set my title element to say exactly that.

The title element must go inside the head element.

The title element is required for all websites as of this writing. It is written as so:

<title>This is the title.</title>

The title element contributes to SEO, as well. It will show up in search engine results. The title also shows up in the favorites section for users who bookmark the page.

You cannot have more than one title element in an HTML document, and if you omit the title element, a site will not work.

Elements 8 - The Style Element

14

The style element is one of the few elements that generally go inside the head element, but doesn't always have to. The style element is used to define CSS from within an HTML document. A developer can also use the style element for what is called "inline style". All this means is that you can use the style element inside the body of your document as well.

What you must know:

The style element defines styling from within an HTML document.
Multiple style elements can exist within a single HTML document.
The style element supports Global Attributes as well as Event Attributes.

Those of you who went through the list of Global Attributes probably noticed the style attribute listed there. The attribute is not exactly the same as the element. The style attribute can be used like the class attribute in the declaration of an element like so:

<p style="border: 1px solid red;">This is a paragraph.</p>

In this instance, the paragraph element will be bordered by a solid red border that is 1px wide. This process is called inline styling.

Most experienced developers rarely rely on inline styling. This touches on a common programming concept called encapsulation. The idea is to have all of your syntax regarding a certain topic in a specific location. A developer would much rather have style syntax in a style sheet so that if a change is needed, the change can be made in a single place, rather than in multiple places on multiple documents.

Those of you familiar with the military or government work call this separation of duties. The ideas are all the same. One person, one job. One document, one job. The HTML document should contain the HTML. The CSS document should contain the CSS. And the Javascript document should contain the Javascript. Writing code inline detracts from this goal, and is commonly considered bad practice.

Elements 9 - The Link Tag

16

First of all, I should make sure you note that this card is named "link tag" not "link element". The reason for this is because the link tag does not have a close. Or, rather, the link tag is self-closing.

The next card will cover all of the self closing tags and how that works in depth, so let's back up to the link tag for a moment.

The link tag is another tag that must be placed within the head tag. The link tag is responsible for linking to an internal or external document that your document relies on. This practice commonly links an HTML document to a CSS or JS document. An internal link reference points to a document that is internal to your project. This would be if the CSS file for your website was in the same directory as your HTML document.

For instance, let's assume your HTML document named "website.html" was located in the "Test" folder on your C: drive. The path to your HTML document would be: "C:\Test\website.html". If you had a stylesheet (CSS document) named "style.css" in a folder called CSS that is inside the Test folder, the path to your stylesheet would be: "C:\Test\CSS\style.css". You can link to this stylesheet with the link tag within the head element of your website.html file like this:

<link rel="stylesheet" href="../CSS/style.css">

This will link the style.css stylesheet to your HTML document for processing by the browser. Whatever CSS syntax in the styles.css document will be processed by the browser and applied to your website.html document on load.

You can reference external stylesheets the same way by just replacing the href attribute of the style tag with the URL for the stylesheet like so:

<link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/bootstrap/4.3.1/css/bootstrap.min.css">

The syntax above will pull in the Boostrap external stylesheet for your website.
This syntax has been simplified from the suggested syntax from the bootstrap website.

The concept seems difficult, but it really isn't. For you, all you need to know is:
If you need to use a stylesheet that you created and put in a folder on your computer, use the first syntax example listed here.
If you need to use an external stylesheet, get the URL for that stylesheet and replace the href attribute of the link tag with that URL as shown in the second example.

You can read more about the link tag here.

Elements 10 - The Meta Tag

16

Metadata is data about data. That sounds dumb, I know, but it is what it is. There is a whole field devoted to data so this concept isn't new. The metadata Wikipedia page can provide a ton of information on the concept of metadata. You're welcome to brief yourself on that topic.

For our purposes, all you need to understand is that the meta tag is exactly what the definition says: data about your data. Things such as the page description, keywords that are associated with the page, and who the author is are stored within meta tags. The w3schools page for the meta tag lays this out well.
