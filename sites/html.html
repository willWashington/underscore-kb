Elements 1 - Elements vs Tags

Now that you know a little about layouts, you can start to learn what elements make up a website. We'll start with the base elements that every website must have and grow from there.

What is an element? Is an element a tag?

An element is the entire statement. An opening tag, the internal items, and a closing tag makes up an element. This is a full element:

<h1 class="large-text">underscore_kb</h1>

A tag is only the opening or closing section of an element. These are the tags in the above elements:

<h1> </h1>

Elements 2 - Doctype

The first element we need to discuss is the doctype element. But, this isn't actually an element.

<!DOCTYPE html>

The doctype element is actually a declaration. By using this statement, you are telling the browser that type of document it is reading.

You can read more about the doctype declaration here. If you are using XHTML, you must declare a proper doctype declaration to instruct the browser of the special intention. For most websites, unless you plan on using XHTML specifically, you are free to use the html declaration, as shown above.

Elements 3 - The Comment

The comment element is used to document code. It is completely ignored when the browser processes the document.

<!--This is a comment.-->

This tag is used to add a comment to the source code. The browser processing the document will completely ignore the comment. Developers use comments to explain code. They are often used to keep up with what a developer was working on when they had to step away.

Comments are useful when debugging code. Commenting out a piece of code can help you understand exactly what that piece of code is doing. For instance, if you have a paragraph element that has a larger font size than you'd expect, you can comment out all of the CSS that is changing font size one by one until you figure out which is causing the change.

The only other thing you need to know about commenting is that you can use it to to hide scripts from browsers without support for scripts. I was not aware of this prior to this writing. You can use the comment tag to "hide" script syntax from the browser in case the browser doesn't have the ability to process the script. This is shown below.

<script type="text/javascript"> <!-- function displayMsg() { alert("Hello World!") } //--> </script>

Elements 4 - The HTML Tag

The HTML tag is pretty clear cut. It tells the browser that the document is an HTML document. It represents the root of an HTML document. This means that it is the beginning of the document. It should wrap all the other elements on your website except the DOCTYPE declaration. The HTML tag should always follow the DOCTYPE declaration like this:

<!DOCTYPE html> <html> ...other elements... </html>

Any div, p, or span tags, and all other tags, should be inside the html tag. The only exception to this rule is the DOCTYPE declaration. It will always be outside the html tag, as shown above.

The html tag is also used for XHTML. I don't fully understand this, so I won't pretend to here for now. You can read more about it here if you're into XHTML. Yes. This is a cop out. I'm hoping to be able to tell you more about XHTML one day.

The html tag can be styled with CSS. It may feel tempting to assign a background color to the html tag with CSS, but this isn't a good idea. My mentors and the tutorials I've watched say this is a common rookie mistake. You often don't want your entire document to have a background color, but rather you want to build an element inside the HTML tag that represents the background. More on this later.

For now, all you really need to know is that browsers have default styles for the html tag that may cause things to not work the way you expect them to. You can reset these styles in your CSS document. This topic is better suited for the CSS page, so I'm going to leave it at that.

Elements 5 - The Head Tag

The Head element is a container for elements that must go inside the head.

Elements that go inside a head element. <title></title> (required in an HTML document) <style></style> (not required to be in the head) <base> <link> <meta> <script></script> (not required to be in the head) <noscript></noscript>

The head element was required in HTML 4.01, but in HTML5 it can be left out. For the most part, all you need to know is that you put the above elements inside the head element. You will almost definitely do this with the link and style tags. Title is required and must be in the head tag, so you'll definitely do that.

Elements 6 - The Body Tag

The body tag encapsulates all of the display elements of the website. All of your images, links, paragraphs, articles, navigation buttons, etc will be inside the body element.

The default CSS for most browsers for the body tag is:

body { display: block; margin: 8px; } body:focus { outline: none; }
